Prototypy

Każda funckja w JavaScript posiada swój prototyp.

Prototyp jest to obiekt definiujący funkcje należące do konkretnego typu obiektu - podobnie jak w przypadku klas w innych językach programowania.

Ale w przypadku prototypów nalepiej przejść od razu do przykładów.

Operacje na tablicach

Obiekt typu Array posiada w swoim prototypie wiele metod przydatych do wykonywnia opreacji na danych.

Każda taka funkcja w domyśle przyjmuje, wspomniany już callback, który może posiadać do trzech argumentów:

  1. Aktualnie przetwarzana wartość
  2. Index na któym znajduje się wartość
  3. Tablica na której wykonywana jest operacja

Podstawowe funckje na tablicach

        
        const data = [1,2,3,4];
        data.forEach((val,index) => consol.log(`Value ${val}, is on index ${index}));
        

        const data = [1,2,3,4];
        data.map((val,index) => val * index); // ???// [0, 2, 6, 12]
    

        const data = [{}, '', 15, null];
        const callback = (val) => typeof val === "number" || typeof val === "undefined";
        data.filter(callback); // ???// [15]
    

        const data = ["Java", "JavaScript", "StarterKit", "Capgemini"];
        data.find(val => val.length > 4); // ???//  "JavaScript"

Biblioteka JEST

JEST to bezkonfiguracyjny freamwork do testowania kodu napisanego w technologiach opartych o JavaScript.

Gdzie umieszczamy testy?

  • Obok pliku testowanego. Plik z testami jest tej samej nazwy co plik testowany lecz z rozszerzeniem .test.js
  • W folderze katalogu __TESTS__
  • W osobnym folderze z testami (obok folderu src - jak w Java)

Z czego składa się plik testami?

Projekt i zaliczenie

Konfiguracja projektu

  1. Pobieramy i instalujemy Node.js ze strony głównej lub za pomoca NVM
  2. Pobieramy template projektu z repo, jako zip.
  3. Wypakowujemy w miejscu projektu i wpisujemy w konsoli npm install.
  4. W celu startu aplikacji developerskiej wpisujemy w konsoli npm start
  5. W celu zbudowania aplikacji produkcyjnej wpisujemy w konsoli npm run build
